# Snake Game — User Feature Research Report

## Gameplay Features
| Feature | User Demand | Priority | Notes |
|---------|------------|----------|-------|
| Speed levels (slow → fast) | Very high | HIGH | Most common request; accommodates casual + competitive |
| Wall wrap-around mode | High | HIGH | "No walls" mode is heavily requested as beginner-friendly option |
| Obstacle mode | Medium | MED | Static walls/blocks on the grid add difficulty progression |
| Multiple food items | Medium | MED | 2–3 food items at once speeds up games, popular with mobile users |
| Timed challenge mode | Medium | MED | "Score as much as possible in 60s" — drives replays |
| Pause functionality | High | HIGH | Critical for mobile; often missing from clones |
| Snake length display | Low | LOW | Niche but requested by completionists |

## UX & Visual Features
| Feature | User Demand | Priority | Notes |
|---------|------------|----------|-------|
| Dark mode / retro green theme | Very high | HIGH | Nokia nostalgia is the #1 emotional hook |
| Color themes / skins | High | MED | Neon, pastel, grayscale — low-effort high-delight |
| Sound effects (toggle) | Medium | MED | Eat sound + game over; must be off by default (mobile) |
| Smooth animation (interpolation) | Medium | MED | Tile-by-tile feels choppy to modern users |
| Mobile d-pad controls | Very high | HIGH | >60% of casual game traffic is mobile |
| Swipe gesture support | High | HIGH | More natural than d-pad on mobile |

## Engagement & Retention Features
| Feature | User Demand | Priority | Notes |
|---------|------------|----------|-------|
| Local high score (localStorage) | Very high | HIGH | #1 retention driver — "beat your best" loop |
| All-time best score display | High | HIGH | Simple but powerful; free to implement |
| Streak counter (daily visits) | Medium | MED | Encourages bookmarking / return visits |
| Achievement badges | Medium | MED | "Score 50", "Survive 2 min" — gamification layer |
| Share score button | Medium | MED | Social sharing drives organic acquisition |
| Global leaderboard | Low | LOW | Requires backend; out of scope v1 |

## Monetization-Friendly Features
| Feature | Compatibility with Ads | Notes |
|---------|----------------------|-------|
| Game over interstitial pause | High | Natural moment to view ads without interrupting play |
| "New game" button placement | High | Place near ad unit to increase viewability |
| How-to-play section | High | Adds page content, improves AdSense eligibility |
| Speed selector UI | Neutral | Adds dwell time before game starts |
